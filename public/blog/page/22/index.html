Prioritizing Network Connections in Mac OS X Snow Leopard

Nov 28th, 2010

Sometimes it is necessary to connect your Mac to multiple networks at the same time, but which one takes priority? I just recently moved and my new place doesn’t have Internet yet. Fortunately for me, I can use my iPhone as a modem until the cable company arrives to install my high speed access.

I connect my notebook to my iPhone and presto, I’m online. Next I turn on my Airport and connect to my local LAN and boom, I’m offline. Even though my Mac sees both my iPhone and my Airport as network devices, it gives priority to the Airport network which is not connected to the Internet. I wish it was possible to prioritize which connection it uses for the Internet.

While snooping around in my Preferences I found this little option: I go ahead and select the “Set Service Order” option and get this dialog: Now my iPhone is the highest priority and all Internet traffic is flowing through it. I can also access my LAN if needed. Problem solved.

How to Secure Your SSH Server.

Nov 17th, 2010

If you are running any variant of Unix or Linux remotely you are probably using SSH to access that system.  This article from Rackspace does a great job telling you how to secure your box so that hackers have a really hard time breaking into your system.

    iptables -A INPUT -j ACCEPT -p tcp --dport 22 -s 10.0.0.20
iptables -A INPUT -j ACCEPT -p tcp --dport 22 -s 10.0.0.25
iptables -A INPUT -j DROP -p tcp --dport 22

Write Your Resume in Markdown, Keep It in Git

Nov 16th, 2010

I am probably not alone in admitting that I do not keep my resume up to date. I do a pretty good job of keeping my LinkedIn profile up to date, but I can’t really email that out as a resume replacement when applying for a job.  If I needed my resume today, I’d grab some old copy lying around on my computer buried in some old folder and re-write it on the spot with what has changed since I last needed it. My memory isn’t so good so chances are good that I will fudge the details.  

I recently read an article on Hacker News about a guy who wrote his resume in Markdown and then published it to the Git source control system where he keeps it up to date.

I think this is overkill, but I like the idea and it certainly does make the chore of resume management a lot easier once you’ve spent the requisite 5 hours setting this up.   The author published a Git project that you can fork if you are a member of Git and then markup to be your own.  If you are a Ruby master, you can easily publish the resume directly to your web-server for all your potential managers/bosses to see.

Simple Tip for Merging Video Files on a Mac (Free)

Nov 16th, 2010

I saw this tip on Mac OS Hints and thought I’d share it.   This is a very easy way to join together two video files into one large file.  I’ve tested it with AVI and MP4 which are both video container files so I can’t say if it works on other file types.

cat video1.avi video2.avi > video1and2.avi

Next open the video in Quicktime X and re-save it. Done.

Using Markdown With Wordpress

Nov 7th, 2010

John Gruber writes a blog called Daringfireball which I read fairly regularly. In order to improve the speed and efficiency of writing blog articles, created a syntax language for writing HTML formatted documents called Markdown. (Read more about Markdown He also wrote a basic parser which will convert the Markdown document into HTML.

The big advantage of Markdown is that it is easier to write than HTML code because it does not require end tags and the syntax is simpler. You will however need some kind of converter to translate the Markdown into HTML in order to gain any sort of benefit of writing in it. For me, the goal was a more powerful way to write my blog articles in Wordpress without resorting to HTML. To do this, I found a good Markdown to HTML plugin for Wordpress and fired up Textmate (my editor of choice) to compose my blog articles. Textmate has built in support for working with Markdown.

Markdown code in Textmate

Mars Edit 3 is the best WYSIWYG blog editor for the Mac in my opinion. It also has support for Markdown, but it will not do syntax highlighting as BBEDIT for Textmate will. It will also convert your Markdown into HTML if you switch the editor from HTML view to Rich Text view. You will need to select the Markdown text filter in the preview window to get the preview to format correctly.

Select the Markdown preview filter to get your formatting to look right

Mars Edit will let you select an external editor to edit your blog posts so if you choose Textmate you can write your articles in Textmate and then post them with Mars Edit.

Mars Edit preferences with Textmate selected as the external editor

It sounds like a lot of work just to get your blog article published and for most people it is, but if you are writing technical articles that have code, formatting, or lots of images, Markdown can really save you time in the long run.
