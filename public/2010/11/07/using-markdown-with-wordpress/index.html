Using Markdown With Wordpress

Nov 7th, 2010

John Gruber writes a blog called Daringfireball which I read fairly regularly. In order to improve the speed and efficiency of writing blog articles, created a syntax language for writing HTML formatted documents called Markdown. (Read more about Markdown He also wrote a basic parser which will convert the Markdown document into HTML.

The big advantage of Markdown is that it is easier to write than HTML code because it does not require end tags and the syntax is simpler. You will however need some kind of converter to translate the Markdown into HTML in order to gain any sort of benefit of writing in it. For me, the goal was a more powerful way to write my blog articles in Wordpress without resorting to HTML. To do this, I found a good Markdown to HTML plugin for Wordpress and fired up Textmate (my editor of choice) to compose my blog articles. Textmate has built in support for working with Markdown.

Markdown code in Textmate

Mars Edit 3 is the best WYSIWYG blog editor for the Mac in my opinion. It also has support for Markdown, but it will not do syntax highlighting as BBEDIT for Textmate will. It will also convert your Markdown into HTML if you switch the editor from HTML view to Rich Text view. You will need to select the Markdown text filter in the preview window to get the preview to format correctly.

Select the Markdown preview filter to get your formatting to look right

Mars Edit will let you select an external editor to edit your blog posts so if you choose Textmate you can write your articles in Textmate and then post them with Mars Edit.

Mars Edit preferences with Textmate selected as the external editor

It sounds like a lot of work just to get your blog article published and for most people it is, but if you are writing technical articles that have code, formatting, or lots of images, Markdown can really save you time in the long run.
